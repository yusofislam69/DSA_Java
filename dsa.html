🤔 What are Data Structures?

Think of data structures as different ways to organize and store information in a computer, just like how you organize things in your room:

  • Array: Like a row of numbered boxes where you can store items
  • Stack: Like a stack of plates - you can only add/remove from the top
  • Queue: Like a line at a store - first person in line gets served first
  • Tree: Like a family tree with parents and children
  • Graph: Like a map showing cities connected by roads

🧠 What are Algorithms?

An algorithm is simply a step-by-step recipe to solve a problem. Just like following a cooking recipe:

  1. Input: The ingredients (your data)
  2. Process: The cooking steps (your algorithm)
  3. Output: The final dish (your result)

💡 Why Learn DSA?

Data Structures and Algorithms are the foundation of programming. They help you:

  • Write faster and more efficient code
  • Solve complex problems step by step
  • Pass technical interviews at top companies
  • Think like a professional programmer

⏱️ Understanding Time Complexity (Big O Notation)

Time complexity tells us how long an algorithm takes to run as the input size grows. Think of it like this:

📊 Common Time Complexities (From Best to Worst)

O(1) - Constant
Like finding a book when you know the exact shelf number
O(log n) - Logarithmic
Like finding a word in a dictionary by opening to the middle
O(n) - Linear
Like reading every page of a book to find a word
O(n²) - Quadratic
Like comparing every student with every other student in class

💾 Understanding Space Complexity

Space complexity tells us how much extra memory an algorithm uses. It's like asking "How much extra storage space do I need?"

// Let's see time complexity in action with simple examples public class TimeComplexityExamples { // O(1) - Constant Time: Always takes the same time public static int getFirstElement(int[] numbers) { return numbers[0]; // Just grab the first element } // O(n) - Linear Time: Time grows with input size public static int findLargestNumber(int[] numbers) { int largest = numbers[0]; // We need to check every number - that's why it's O(n) for (int i = 1; i < numbers.length; i++) { if (numbers[i] > largest) { largest = numbers[i]; } } return largest; } // O(n²) - Quadratic Time: Nested loops public static void printAllPairs(int[] numbers) { // For each number... for (int i = 0; i < numbers.length; i++) { // ...compare it with every other number for (int j = 0; j < numbers.length; j++) { System.out.println("Pair: " + numbers[i] + ", " + numbers[j]); } } } // Let's test our functions public static void main(String[] args) { int[] myNumbers = {5, 2, 8, 1, 9}; System.out.println("First element: " + getFirstElement(myNumbers)); System.out.println("Largest number: " + findLargestNumber(myNumbers)); } }

⚠️ Important for Beginners

Don't worry if Big O notation seems confusing at first! As you practice more problems, it will become natural. Focus on understanding the concept: "How does the algorithm's performance change as we give it more data?"

🤔 Array vs ArrayList - When to Use Which?

  • Use Array when: You know the exact size and it won't change
  • Use ArrayList when: Size might change, you need to add/remove elements
  • Performance: Arrays are slightly faster, but ArrayList is more flexible

// Solution 1: Brute Force - O(n²) time, O(1) space public static int[] twoSumBruteForce(int[] nums, int target) { System.out.println("Using brute force approach..."); // Check every pair of numbers for (int i = 0; i < nums.length; i++) { for (int j = i + 1; j < nums.length; j++) { System.out.println("Checking: " + nums[i] + " + " + nums[j] + " = " + (nums[i] + nums[j])); if (nums[i] + nums[j] == target) { System.out.println("Found solution!"); return new int[] {i, j}; } } } return new int[] {}; // No solution found } // Solution 2: Hash Map - O(n) time, O(n) space public static int[] twoSumOptimal(int[] nums, int target) { System.out.println("Using hash map approach..."); HashMap map = new HashMap<>(); for (int i = 0; i < nums.length; i++) { int complement = target - nums[i]; System.out.println("Looking for complement of " + nums[i] + ": " + complement); if (map.containsKey(complement)) { System.out.println("Found complement at index " + map.get(complement)); return new int[] {map.get(complement), i}; } map.put(nums[i], i); System.out.println("Added " + nums[i] + " at index " + i + " to map"); } return new int[] {}; // No solution found }
2. Best Time to Buy and Sell Stock (LeetCode #121)
Easy

public static int maxProfit(int[] prices) { if (prices.length < 2) { return 0; // Can't make profit with less than 2 prices } int minPrice = prices[0]; // Lowest price seen so far int maxProfit = 0; // Maximum profit found so far System.out.println("Day 0: Price = " + prices[0] + ", Min Price = " + minPrice + ", Max Profit = " + maxProfit); for (int i = 1; i < prices.length; i++) { int currentPrice = prices[i]; // Calculate profit if we sell today int todayProfit = currentPrice - minPrice; // Update maximum profit if today's profit is better if (todayProfit > maxProfit) { maxProfit = todayProfit; } // Update minimum price if today's price is lower if (currentPrice < minPrice) { minPrice = currentPrice; } System.out.println("Day " + i + ": Price = " + currentPrice + ", Min Price = " + minPrice + ", Today's Profit = " + todayProfit + ", Max Profit = " + maxProfit); } return maxProfit; }

// Node class - the building block of linked lists class ListNode { int val; // The data stored in this node ListNode next; // Reference to the next node // Constructors ListNode() {} ListNode(int val) { this.val = val;
